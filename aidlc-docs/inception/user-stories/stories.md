# User Stories

구성 방식: User Journey-Based | 형식: As a... | 인수 기준: 체크리스트 | 세분화: 기능 단위

---

## 고객 Journey

### US-C-01: 테이블 자동 로그인

**As a** 고객으로서,  
**I want** 테이블에 앉으면 별도 로그인 없이 주문 화면이 바로 열리기를,  
**So that** 즉시 메뉴를 탐색하고 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 관리자가 1회 매장 식별자, 테이블 번호, 비밀번호를 설정하면 로컬에 저장된다
- [ ] 이후 페이지 접근 시 저장된 정보로 자동 인증되어 메뉴 화면이 표시된다
- [ ] 자동 로그인 실패 시 설정 화면으로 이동한다

---

### US-C-02: 메뉴 탐색

**As a** 고객으로서,  
**I want** 카테고리별로 정리된 메뉴를 카드 형태로 탐색하기를,  
**So that** 원하는 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 화면이 기본(홈) 화면으로 표시된다
- [ ] 카테고리 탭을 선택하면 해당 카테고리 메뉴만 표시된다
- [ ] 각 메뉴 카드에 메뉴명, 가격, 설명, 이미지가 표시된다
- [ ] 모든 버튼은 최소 44x44px 이상이다

---

### US-C-03: 장바구니 관리

**As a** 고객으로서,  
**I want** 메뉴를 장바구니에 담고 수량을 조절하기를,  
**So that** 주문 전에 원하는 구성을 자유롭게 변경할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 카드에서 담기 버튼을 누르면 장바구니에 추가된다
- [ ] 장바구니에서 수량 증가/감소 및 항목 삭제가 가능하다
- [ ] 총 금액이 수량 변경 즉시 실시간으로 업데이트된다
- [ ] 페이지 새로고침 후에도 장바구니 내용이 유지된다

---

### US-C-04: 주문 생성

**As a** 고객으로서,  
**I want** 장바구니 내용을 확인하고 주문을 확정하기를,  
**So that** 주방에 정확한 주문이 전달된다.

**Acceptance Criteria:**
- [ ] 주문 확정 전 주문 내역(메뉴, 수량, 총액)을 확인할 수 있다
- [ ] 주문 성공 시 주문 번호가 표시되고 장바구니가 비워진다
- [ ] 주문 성공 후 메뉴 화면으로 자동 이동한다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다

---

### US-C-05: 주문 내역 조회

**As a** 고객으로서,  
**I want** 현재 테이블에서 주문한 내역을 확인하기를,  
**So that** 어떤 메뉴를 주문했는지, 현재 상태가 어떤지 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
- [ ] 각 주문에 주문 번호, 시각, 메뉴 및 수량, 금액, 상태가 표시된다
- [ ] 주문 상태는 대기중 / 준비중 / 완료 중 하나로 표시된다
- [ ] 주문 시간 순으로 정렬된다

## 관리자 Journey

### US-A-01: 매장 로그인

**As a** 매장 관리자로서,  
**I want** 매장 식별자와 계정으로 로그인하기를,  
**So that** 내 매장의 주문 및 운영 데이터에만 접근할 수 있다.

**Acceptance Criteria:**
- [ ] 매장 식별자, 사용자명, 비밀번호 입력으로 로그인된다
- [ ] 로그인 성공 시 JWT 토큰이 발급되고 16시간 세션이 유지된다
- [ ] 브라우저 새로고침 후에도 세션이 유지된다
- [ ] 16시간 경과 후 자동 로그아웃된다
- [ ] 로그인 시도 횟수 초과 시 일시적으로 잠긴다

---

### US-A-02: 실시간 주문 모니터링

**As a** 매장 관리자로서,  
**I want** 들어오는 주문을 테이블별로 실시간으로 확인하기를,  
**So that** 주문 누락 없이 빠르게 대응할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블별 카드 그리드 레이아웃으로 주문 현황이 표시된다
- [ ] 신규 주문이 2초 이내에 화면에 반영된다 (SSE)
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- [ ] 카드 클릭 시 해당 테이블의 전체 주문 상세 내역이 표시된다
- [ ] 주문 상태를 대기중 / 준비중 / 완료로 변경할 수 있다
- [ ] 신규 주문 발생 시 시각적 강조(색상 변경 또는 애니메이션)가 표시된다

---

### US-A-03: 테이블 관리

**As a** 매장 관리자로서,  
**I want** 테이블 초기 설정, 주문 삭제, 이용 완료 처리, 과거 내역 조회를 하기를,  
**So that** 테이블 세션 라이프사이클을 효율적으로 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블 번호와 비밀번호를 설정하면 16시간 세션이 생성된다
- [ ] 특정 주문 삭제 시 확인 팝업이 표시되고, 확인 후 즉시 삭제 및 총액이 재계산된다
- [ ] 이용 완료 처리 시 확인 팝업이 표시된다
- [ ] 이용 완료 처리 후 현재 세션 주문이 과거 이력으로 이동하고 테이블 총액이 0으로 리셋된다
- [ ] 과거 내역 버튼 클릭 시 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다
- [ ] 과거 내역에서 날짜 필터링이 가능하다

---

### US-A-04: 메뉴 관리

**As a** 매장 관리자로서,  
**I want** 메뉴를 등록, 수정, 삭제하고 순서를 조정하기를,  
**So that** 매장 상황에 맞게 메뉴판을 최신 상태로 유지할 수 있다.

**Acceptance Criteria:**
- [ ] 카테고리별 메뉴 목록을 조회할 수 있다
- [ ] 메뉴명, 가격, 설명, 카테고리, 이미지 파일을 입력하여 메뉴를 등록할 수 있다
- [ ] 기존 메뉴의 정보를 수정할 수 있다
- [ ] 메뉴를 삭제할 수 있다
- [ ] 메뉴 노출 순서를 조정할 수 있다
- [ ] 필수 필드 누락 또는 잘못된 가격 입력 시 유효성 검사 오류가 표시된다
